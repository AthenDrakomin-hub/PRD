# KForge | 墟 - 笔记模板与自动化脚本构建计划

## 摘要
根据用户提供的完整内容，在项目中创建一套用于知识库管理的标准化笔记模板、标签词典、灵感收集箱，以及用于网络攻防模拟实验的自动化脚本框架和目录结构。

## 当前状态分析
目前项目根目录下主要包含前端代码（`src/`）、之前的 markdown 文档（`docs/`）以及各类配置文件。尚未建立 `02_网络隐匿/`、`simulations/` 和 `scripts/` 目录，也缺少标准化的笔记模板和自动化脚本。

## 提议的变更
将按要求创建以下目录和文件：

1. **根目录基础文件**
   - `TEMPLATE.md`：完整的笔记模板，包含 Front Matter 和标准章节结构。
   - `TAGS.md`：标签词典，用于维护统一的标签体系。
   - `INBOX.md`：待处理灵感收集箱。

2. **笔记内容目录及文件**
   - 创建目录 `02_网络隐匿/`。
   - 在该目录下创建 `隐匿_透明代理-TPROXY实战.md`，作为完整示例笔记。

3. **模拟实验目录及文件**
   - 创建目录 `simulations/template/`（使用 `template` 作为占位场景名）。
   - 在该目录下创建 `scenario.md`，作为模拟场景定义文件模板。

4. **自动化脚本目录及文件**
   - 创建目录 `scripts/`。
   - 在该目录下创建 `run_sim.sh`：用于启动 Docker 环境、执行攻防脚本并生成实战笔记。
   - 在该目录下创建 `generate_note.py`：配合 Bash 脚本，用于从模拟环境日志自动生成结构化笔记。

## 假设与决策
- **场景目录命名**：对于第五项的 `simulations/[场景名]/scenario.md`，决定创建一个名为 `template` 的子目录作为占位，即将文件放置在 `simulations/template/scenario.md` 中。
- **文件权限**：在实现阶段，将会为 `scripts/run_sim.sh` 和 `scripts/generate_note.py` 赋予可执行权限（`chmod +x`）。

## 验证步骤

2. 检查所有 7 个文件是否已按要求生成，且内容完整无误。
3. 检查脚本文件（`.sh` 和 `.py`）是否具有执行权限。
