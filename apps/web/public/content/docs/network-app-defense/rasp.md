# 运行时应用自我保护 (RASP)

## 核心作用
将安全防护彻底下沉到代码执行层，从应用程序内部直接监控和拦截高级攻击，是纵深防御的最后一道防线。

## 实现原理
通过语言级别的探针（如 Java Agent, Node.js Hooks, PHP 扩展），在代码执行到底层关键函数（如 `Runtime.exec`、数据库 `SQL execute`、文件 `open`）前进行劫持与安全检查。由于 RASP 掌握着最完整的上下文和最终的解密变量值，它能极其精准地判断该操作是否为攻击。

## 主要防御手段
- 无视任何网络层的编码混淆，直接看透 Payload 本质。
- 精准拦截反序列化漏洞、0-day RCE 和极度复杂的 SQL 注入。

## 漏洞与局限
- **语言强相关**：需要为每种编程语言甚至不同版本单独开发探针，部署与维护极其困难。
- **业务稳定性风险**：探针直接嵌入应用进程，一旦 RASP 出现 Bug，将直接导致业务进程崩溃或产生严重的性能损耗。
- **可能与 APM 工具冲突**：常与 SkyWalking 等性能监控工具争夺字节码插桩权限。

## 典型工具
- **OpenRASP**: 百度开源的优秀 RASP 解决方案。
- **Tetragon**: Cilium 团队基于 eBPF 打造的系统级 RASP 平替，在内核态实现无侵入防护。

## 实践建议
在一个易受攻击的 Java Web 靶场（如 WebGoat）中挂载 OpenRASP agent，尝试执行 OGNL 表达式注入，观察 RASP 是如何在方法调用前瞬间完成阻断并记录日志的。
