# 咋让电脑所有流量都走代理？（小白友好版·注解版）

## 你碰到啥情况了？
- 你开了个梯子，浏览器能翻墙了。
- 但是你打开命令行，敲个 `ping google.com` 不通，或者 `curl ip.sb` 显示的还是你真实 IP。
- 因为你那个梯子只管浏览器，命令行里的程序压根没走代理。

## 咋解决？
让电脑上**所有程序**的流量，不管它自己愿不愿意，都乖乖走代理。
这个技术叫 **透明代理** (TPROXY)。

## 手把手教你弄（用 Docker 练手，不搞坏电脑）

### 第一步：找个支持透明代理的梯子客户端
我们用一个叫 Clash 的软件，它有个 Docker 版本，不用安装直接跑。

打开你的 Linux 终端（没有 Linux 就用虚拟机装一个 Ubuntu），输入：
```bash
# 这条命令会从网上下载并运行 Clash 的 Docker 镜像
# --cap-add=NET_ADMIN 是给它修改网络的高级权限（搞透明代理必须的）
# -v ~/clash:/root/.config/clash 是把我们本地的配置文件夹挂载进容器，方便我们随时改配置
# -p 7890:7890 是把容器里的 7890 端口映射出来，方便其他程序连接
docker run -d --name my-clash \
  --cap-add=NET_ADMIN \
  -v ~/clash:/root/.config/clash \
  -p 7890:7890 \
  dreamacro/clash
```

### 第二步：给这个 Clash 加点配置文件
在你电脑的家目录下建个文件夹叫 `clash`，里面放一个文件叫 `config.yaml`。内容大概是：
```yaml
# 这是 Clash 的配置文件，你把自己的服务器信息填进去就行
proxies:
  - name: "我的节点"
    type: socks5
    server: 你服务器的IP
    port: 你服务器的端口
    # 如果需要密码就加上 username 和 password

tproxy: true          # 这行是重点！打开透明代理模式
tproxy-port: 7890
```

### 第三步：进到 Clash 容器里，敲几条命令
```bash
# 进入容器
docker exec -it my-clash bash

# 开启路由转发（让流量能通过这台机器）
# 说白了就是告诉 Linux：“允许数据包从你这儿经过，去往别的地方”
echo 1 > /proc/sys/net/ipv4/ip_forward

# 搞个路由表，等下用
# 这条命令的意思是：所有带着标记“1”的数据包，都给我用第100号路由表来处理
ip rule add fwmark 1 table 100
# 这条命令的意思是：在100号路由表里，不管去哪儿的流量，都给我发到本地回环网卡（lo）上，这样代理程序才能“看见”它们
ip route add local 0.0.0.0/0 dev lo table 100

# 加几条防火墙规则，把流量劫持到 Clash
# 新建一条叫 DIVERT 的链，用来处理“已经被代理过”的连接，防止数据包进入死循环
iptables -t mangle -N DIVERT
# 给进入 DIVERT 链的数据包打上标记“1”
iptables -t mangle -A DIVERT -j MARK --set-mark 1
# 然后放行这些数据包
iptables -t mangle -A DIVERT -j ACCEPT
# 这条规则检查数据包是不是属于一个已经存在的连接，如果是，就扔到 DIVERT 链去处理
iptables -t mangle -I PREROUTING -p tcp -m socket -j DIVERT

# 核心规则：劫持所有出去的 TCP 流量，打上标记 1，并交给 TPROXY
# TPROXY 是 iptables 的一个扩展，能把流量“透明地”重定向到代理程序
iptables -t mangle -A PREROUTING -p tcp -j TPROXY --on-port 7890 --tproxy-mark 0x1/0x1

# 搞定，退出容器
exit
```

### 第四步：测试一下效果
找个别的容器，借用 `my-clash` 的网络：
```bash
# 这条命令的意思是启动一个新容器，但它的网络配置完全照搬 my-clash 容器的
docker run --rm -it --network container:my-clash alpine sh
```
现在你在新容器的命令行里，敲：
```bash
apk add curl
curl `http://ip.sb`
```
如果显示的 IP 是你梯子服务器的 IP，那就成了！这个容器里的所有流量都走代理了，不管你用啥命令。

## 常见问题
- **curl 没反应，超时**：检查下 Clash 容器里 `netstat -tlnp | grep 7890` 看端口有没有监听。
- **还是显示真实 IP**：大概率是那几条 iptables 命令没敲对，重新敲一遍。
- **提示没有 iptables 命令**：在容器里 `apt update && apt install iptables -y`。

## 看完这个，你可以去研究啥？
- 把容器换成真实的 Linux 机器，一样的方法。
- 想代理 UDP 流量（比如玩游戏）？把命令里的 `tcp` 改成 `udp` 再来一遍。
- 这玩意儿就是所谓的 TPROXY，想知道它和普通 REDIRECT 的区别？等你把这套跑通了再回头看原理。
