# 下一代防火墙 (NGFW)

## 核心作用
结合传统 L4 防火墙与深度包检测 (DPI) 技术，实现细粒度的应用层可见性与管控，不仅看端口，更看懂“应用”。

## 实现原理
在 L3/L4 基础上，NGFW 重组 TCP 数据流，解析 L7 协议（如 HTTP, DNS, TLS），并结合海量的应用指纹库与实时威胁情报，实现精细化管控（例如：“允许使用微信聊天，但拦截微信文件传输”）。

## 主要防御手段
- 恶意应用与代理协议封堵。
- 基于身份（而非仅 IP）的网络微隔离。
- 端口/协议异常检测（如在 80 端口跑 SSH）。

## 漏洞与局限
- **加密流量盲区**：面对全量 TLS 1.3 / ECH 加密流量，若不进行破坏性的 SSL 卸载（中间人），L7 审计能力将大幅下降。
- **性能瓶颈**：深度包检测极其消耗 CPU 资源，容易成为高并发网络下的性能瓶颈。

## 典型工具
- **OPNsense / pfSense**: 极其强大的开源防火墙操作系统。
- **Zenarmor**: 基于现代 eBPF 技术构建的开源 NGFW 引擎插件。

## 实践建议
在虚拟机中安装 OPNsense 并启用 Zenarmor 引擎，配置安全策略尝试封堵所有的 P2P 下载流量，并查看拦截日志的可视化面板。
