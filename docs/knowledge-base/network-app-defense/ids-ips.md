# 入侵检测与防御系统 (IDS/IPS)

## 核心作用
实时监控网络流量，发现并即时阻断已知的漏洞利用、扫描行为和恶意网络通信。

## 实现原理
通过旁路镜像流量 (IDS) 或串联部署 (IPS)，利用高性能的规则匹配引擎（如 Aho-Corasick 算法），将抓取到的数据包内容与数万条威胁特征签名（如 Emerging Threats 规则库）进行高速比对。

## 主要防御手段
- 阻断明文传输的 RCE、SQL 注入攻击。
- 识别并切断恶意软件的 C2 (命令与控制) 远控信标。
- 检测内网横向移动与大范围端口扫描。

## 漏洞与局限
- **误报灾难 (Alert Fatigue)**：极易产生海量误报，耗尽安全运营人员的精力。
- **0-day 无力**：高度依赖已知签名，对未知的 0-day 攻击和强混淆的加密流量基本无能为力。
- **规则维护成本高**：需要持续不断地订阅和调优庞大的规则库。

## 典型工具
- **Suricata**: 支持多线程与 eBPF 加速的现代 IDS/IPS。
- **Zeek (原 Bro)**: 偏向于网络安全监控与复杂元数据行为分析的框架。
- **Snort 3**: 老牌 IDS 的全新升级版。

## 实践建议
部署 Suricata，编写一条自定义规则，专门检测带有特定恶意 User-Agent 的 HTTP 请求，并发起模拟请求触发高危告警。
