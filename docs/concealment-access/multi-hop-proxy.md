# 多级跳板与代理链 (Multi-hop Proxy)

## 核心作用
在网络攻防中隐藏真实源头与目标资产，极大提升追踪溯源成本。通过在链路中串联多个代理节点，使得任何单一节点被攻破都无法还原完整的通信路径。

## 实现原理
流量依次经过多个节点（如 Node A -> Node B -> Node C），每层代理仅知晓其相邻的节点信息。在现代网络架构中，通常结合 TLS/XTLS 等强加密隧道（如 SOCKS5 over TLS 或 VLESS），确保即使流量被中间节点截获，也无法窥探真实载荷，彻底防止中间人嗅探。

## 主要防御手段
- 部署严格的出口 IP 白名单机制。
- 深度包检测 (DPI) 识别未混淆的代理握手特征。
- 部署基于时序和包大小的流量关联分析模型。

## 漏洞与局限
- **网络延迟显著增加**：物理链路变长导致极高延迟，不适用实时性要求高的业务。
- **单点信任风险**：若链路中某关键节点为恶意蜜罐（记录了流量日志），隐匿性可能被彻底破坏。
- **全局分析脆弱性**：易受具备全网视野的全局被动对手 (GPA) 的流量时序分析攻击。

## 典型工具
- **Proxychains-ng**: 经典的 Linux 命令行代理链路由工具。
- **Xray-core**: 现代网络代理内核，支持复杂的路由分流和多协议链式代理。
- **Clash Meta (Mihomo)**: 支持高级策略路由的强大开源代理内核。

## 实践建议
使用 Proxychains 结合本地开启的 SOCKS5 代理（如连接到远端 VPS），尝试在终端运行 `proxychains curl ifconfig.me`，验证出口 IP 是否已发生变化。
